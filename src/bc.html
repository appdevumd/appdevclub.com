Terminal Command Cheatsheet

  • pwd – Print working directory – Shows what folder you currently are in.
  • ls – List directory – Shows the content of the folder you are currently in.
    • ls -a – Includes hidden folders in output.
    • ls -l – Shows more information about each thing in the folder.
    • ls -lh – Shows more information about each thing in the folder and shows file sizes in "human-readable" numbers (like 42M or 7G, instead of long numbers)
  • cd folder – Change directory – Navigates to folder, such that any commands that you use after cd are applied relative to folder. For example, running cd folder and less file is equivalent to running less folder/file.
    • cd .. – Moves to the folder that contains the one that you are currently in. In other words, changes to the parent directory.
    • cd ~ – Returns to your home folder.
    • cd / – Moves to the root folder.
  • cat file – Prints the contents of file to the screen.
  • less file – Opens file in less, which lets you scroll through file and search for things within it. In general, less is better for longer files than cat.
  • mkdir new-folder – Make directory – Creates a new folder in the current one, named new-folder.
  • rm file – Remove – Deletes the file named file, it does not get sent to any sort of "trash," so be careful when using it.
  • rm -rf folder – Remove, recursively, and with force – Deletes the folder named folder and everything in it forever. This is a dangerous command, if you aren't careful, this command will let you obliterate every file on your computer.

Vim Cheatsheet

Vim is a terminal-based text editor. Much like Notepad, TextEdit, or Visual Studio Code, Vim lets you edit any sort of text file. However, it has one major difference: it runs solely inside of a terminal. Vim is also different in another way: Vim is a modal editor, which means that actions like text-editing and things like saving are done from different "modes"

To start editing a file with vim, just run vim file, where file is the name of the text file you want to edit. This will open Vim in its "normal" mode.

"Normal" Mode Commands

You can think of normal mode as the mode from which you run commands and enter other modes. A "full" Vim cheatsheet could be the size of a small novel, so this cheatsheet only lists the most basic commands.

  • i – Switch to "insert" mode, the mode where you can actually type.
  • :wq, then press enter – Save the current file and then quit.
  • :w, then press enter – Save the current file, but don't quit.
  • :q!, then press enter – Quit without saving.

"Insert" Mode

The primary usage of insert mode is typing, so pressing keys do what you would expect in a text editor, it will type! However, to exit insert mode, just press escape and you will be back in normal mode.

An example Vim session

Below is a walkthrough of what making a file might look like in Vim, and a description of what each step does.

  1. vim file.txt – Start editing file.txt.
  2. i – Enter insert mode.
  3. Type Hello world! – Types the text, Hello world! into the new file.
  4. Press escape – Return to normal mode.
  5. :wq, then press enter – Writes the changes made (i.e. the text Hello world!) to file.txt and leave Vim.
